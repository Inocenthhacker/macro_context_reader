## Obiectiv

Industry-standard equilibrium model pentru EUR/USD, inspirat din BBVA Research (Martínez et al. 2025). Răspunde la întrebarea: *"Unde ar trebui să fie EUR/USD vs. unde e, conditional de regimul curent de Global Financial Conditions?"*

PRD-301 livrează un **anchor signal** complementar lui PRD-300 (trend/momentum) — un nivel absolut de referință, nu un trend. Output-ul alimentează PRD-500 ca a 5-a sursă de evidență în fuziunea Dempster-Shafer.

**Distincție conceptuală vs PRD-300:**
- PRD-300 = "unde merge prețul" (trend signal, composite divergence)
- PRD-301 = "unde ar trebui să fie prețul" (level signal, equilibrium misalignment)

## Design principles

1. **Industry standard** — replicăm metodologia BBVA Research 2025 și ECB working papers. Zero invenție de framework.
2. **Regime-conditional equilibrium** — nu un singur echilibru, ci 3 scenarii (central / subdued GFCI / trade tensions), selecție runtime bazată pe Global Financial Conditions proxy.
3. **Decompoziție USD vs EUR** — o mișcare EUR/USD poate fi USD strength (rapid, GFCI-dependent) sau EUR weakness (lent, structural). Ambele corectate diferit. BBVA (2025) documentează explicit ambele.
4. **FRED-native GFCI proxy** — folosim Chicago Fed NFCI (FRED ticker: `NFCI`) ca proxy pentru Goldman Sachs GFCI (care e proprietar). NFCI corelație >0.85 cu GFCI (confirmat academic), disponibil gratuit săptămânal din 1971.

## Cerințe Funcționale

### GFCI Proxy Layer
- [ ] REQ-1: Ingestie NFCI (Chicago Fed National Financial Conditions Index) din FRED — serie săptămânală
- [ ] REQ-2: Interpolare NFCI pe frecvență zilnică (forward-fill între publicări săptămânale)
- [ ] REQ-3: Persistență Parquet: `data/equilibrium/gfci_proxy.parquet`

### Equilibrium Compute Layer
- [ ] REQ-4: 3 scenarii de echilibru hard-coded conform BBVA 2025:
  - `equilibrium_central = 1.20` (GFCI neutru, productivitate normală)
  - `equilibrium_subdued = 1.10` (GFCI stres, risk-off moderat)
  - `equilibrium_trade_tensions = 1.05` (tariffs escaladate, decoupling)
- [ ] REQ-5: Selector runtime: `select_equilibrium(nfci_current) -> equilibrium_level` bazat pe thresholds NFCI calibrate empiric (default: NFCI > 0 → subdued, NFCI > 1 → trade_tensions)
- [ ] REQ-6: Misalignment signal: `misalignment_pct = (eurusd_current - equilibrium_current) / equilibrium_current`

### USD/EUR Decomposition Layer
- [ ] REQ-7: Ingestie US Dollar Index (DXY proxy — FRED ticker `DTWEXBGS` Broad USD Index)
- [ ] REQ-8: Decompoziție: `eurusd_change = usd_contribution + eur_contribution + residual` via regression decomposition (standard BBVA methodology)
- [ ] REQ-9: Output flag: `dominant_driver: Literal["USD_strength", "EUR_weakness", "mixed"]` bazat pe contribuția relativă

### Output Layer
- [ ] REQ-10: Schema Pydantic `EquilibriumSignal(as_of_date, eurusd_current, equilibrium_central, equilibrium_conditional, misalignment_pct, dominant_driver, nfci_current, confidence)`
- [ ] REQ-11: Persistență signal: `data/equilibrium/equilibrium_history.parquet`
- [ ] REQ-12: Interfață consumatoare pentru PRD-500 BBA mapper (descrisă în PRD-500)
